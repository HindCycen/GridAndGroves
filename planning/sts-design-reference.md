## 1. StS 核心设计哲学

### 1.1 Mega Crit 的设计理念

根据 Mega Crit 联合创始人 Anthony Giovannetti 在多个采访和 GDC 演讲中透露的信息：

**"让玩家觉得自己很聪明"**
- StS 的核心不是惩罚玩家，而是给玩家展示聪明决策的机会
- 每一张牌在特定情境下都可能是"正确选择"
- 没有绝对的最优解，只有不同情境下的相对最优

**"可重玩性来自组合的深度，而非内容数量"**
- StS 只有 ~75 张角色卡牌 + 约 50 件遗物
- 但玩家可以玩数百小时——因为卡牌之间、卡牌与遗物之间的交互创造了指数级的可能性
- 设计原则：**机制交互 >> 内容堆砌**

**"每局都是一个新的解谜游戏"**
- 随机的卡牌掉落和遗物迫使玩家每局重新评估策略
- 优秀的 StS 玩家不是"记住了一种通关套路"，而是学会了"如何识别当前局势下的最优路径"

### 1.2 StS 的三个核心设计支柱

| 支柱 | 说明 | 体现 |
|------|------|------|
| **决策密度** | 玩家每个回合都需要做多个有意义的决策 | 出牌顺序、是否留费、是否保留某张牌、走哪条路线 |
| **信息透明度** | 玩家能获取足够信息做出 informed decision | 敌人意图可见、遗物效果明确、卡牌升级后可预览 |
| **非对称性** | 每个角色的玩法差异足够大 | 铁甲兵（力量/消耗）、猎手（毒/小刀）、故障机器人（球/冰）、观者（姿态） |

### 1.3 StS 与 传统 CCG 的本质区别

| 维度 | Hearthstone / MTG | Slay the Spire |
|------|------------------|----------------|
| 对手 | 人类玩家（不可预测） | AI（可预测模式） |
| 牌组构建 | 赛前构筑，赛中不变 | 赛中动态构建，每局不同 |
| 生命值 | 双方公平起始 | 玩家 vs 环境，敌人有固定模式 |
| 资源系统 | 线性增长（每回合+1水晶/地） | 固定 3 费/回合（可通过遗物改变） |
| **失败原因** | **被对手 outplay** | **自己被自己的决策坑了** |

> 这个区别至关重要：StS 玩家输掉后会反思"我该选哪张牌""我该走哪条路线"，而不是"对面运气真好"——这让失败也成为学习体验。

## 2. 卡牌设计原则

### 2.1 卡牌类型体系

StS 将卡牌分为三大类，每类有明确的定位：

| 类型 | 框架 | 定位 | 设计约束 |
|------|------|------|----------|
| **攻击 (Attack)** | 五边形 | 直接伤害 | 必须造成直接伤害，不能是间接伤害 |
| **技能 (Skill)** | 矩形 | 防御/功能 | 不能造成直接伤害（但可以通过 debuff 间接造成） |
| **能力 (Power)** | 圆形 | 永久增益 | 每场战斗每张限一次，打出后从牌组移除 |

**设计规则**：
- 攻击牌可以附带额外效果，但必须有直接伤害
- 技能牌可以做任何事除了直接伤害
- 能力牌不能太强（因为是永久效果），也不能太弱（不然不值得占用一个卡位）

### 2.2 关键词机制设计

StS 的关键词系统是其优雅设计的核心：

| 关键词 | 设计目的 |
|--------|---------|
| **消耗 (Exhaust)** | 控制牌组厚度，创造"一次性强力牌"设计空间 |
| **虚无 (Ethereal)** | 限制强力牌的"囤积"行为——不用就失去 |
| **固有 (Innate)** | 保证核心牌会上手，减少随机性带来的挫败感 |
| **保留 (Retain)** | 给玩家策略延迟执行的能力，增加决策深度 |
| **能力 (Power)** | 永久效果，但占用一个卡位——一次性的长期投资 |

**社区共识**（来自 Reddit r/slaythespire）：
- "Exhaust 是 StS 最好的机制，因为它让玩家有动力精简牌组"
- "Ethereal 是一把双刃剑——它可以防止强力牌变成纯粹的牌组污染，但如果太多会限制玩法多样性"
- "Retain 牌的设计很精妙——它们给玩家 '期权价值'，而不是强制的当下收益"

### 2.3 升级设计

StS 的卡牌升级系统：
- 升级通常增加数值（6→9）或降低费用（2→1）
- 极少改变机制（少数例外：如 `Searing Blow` 可重复升级）
- 升级后的数值提升幅度大约是 **原值的 1.3~1.5 倍**
- 升级不是必须的，但往往是优先选择

**升级设计原则**：
1. 升级后的卡牌不应该改变核心玩法（保持可预测性）
2. 数值提升应该在"值得花一个篝火"的程度
3. 不要所有牌升级都同样强——创造优先级的梯度

### 2.4 "坏牌"的设计价值

StS 的一个独特设计是包含一些**在大多数情况下不那么好的卡牌**：

- `Clash`（只能在全手牌都是攻击牌时打出）
- `Setup`（将一张手牌放到牌堆顶，下回合抽到）
- `Runic Pyramid` 遗物+某些牌的组合

**为什么设计"不那么强的牌"？**
1. **降低初学者的决策负担**——新玩家不必为每张奖励都纠结
2. **创造"发现价值"的快乐**——老玩家知道怎样让"废牌"发光
3. **让稀有卡牌感觉更稀有**——如果张张都是神卡，神卡就不神了
4. **局部最优 ≠ 全局最优**——特定卡组构型下，"废牌"可能是关键组件

## 3. 数值平衡方法论

### 3.1 费用模型

StS 的基础资源模型：

| 费用 | 说明 | 典型伤害 | 典型护盾 |
|------|------|---------|---------|
| 0 | 免费效果，通常很弱或有条件 | 3~4 | 2~4 |
| 1 | 标准费用，大部分卡牌在此 | 6~12 | 5~12 |
| 2 | 强力效果，需要构筑配合 | 12~24 | 15~20 |
| 3 | 核心动作，通常需要减费配合 | 20~35 | 25~30 |
| X | 可变的投入→可变的产出 | 任意 | 任意 |

**费用模板公式**（经验法则）：
- 1 费白板伤害 = 6（打击 Strike）
- 1 费白板护盾 = 5（防御 Defend，实为 5 护盾 × 实为 5）
- 每增加 1 费，伤害约 ×1.8~2.0，护盾约 ×1.8~2.0
- 附带效果（抽牌、debuff、buff）通常从数值中扣减
- 负面效果（自伤、增加状态牌）通常会提供数值补偿（+30%~50%）

### 3.2 抽牌的价值

StS 中抽牌的价值非常高——因为抽到"对的牌"比抽到"打击"强得多：

- 抽 1 张牌 ≈ 0.5~0.75 费的价值
- `Backflip`（1 费 5 护盾 + 抽 2 张）：如果只算护盾价值 ~0.8 费，加上抽牌 ≈ 1.8~2.0 费
- 但抽牌的价值是**情境依赖**的——牌组越强，抽牌越有价值
- 空抽（牌组已空的抽牌）本质上等于无价值

**引申到 G&G**：虽然没有直接的"抽牌"机制，但 G&G 中的"放置格数释放"（松动）、"路径操控"（让 Bot 多次触发 Block）可以作为类比——让玩家更频繁地使用高质量 Block。

### 3.3 AOE（群伤）的价值

- AOE 在 StS 中的价值高于单体，但 AOE 卡牌的单目标效率通常低于单体卡
- 一场战斗中敌人越多，AOE 的价值越高
- 数值大约为单体 70%~80%（对单目标时）
- 但 AOE 的**总价值**（所有目标合计）通常超过单体
- 设计约束：AOE 不应让单体卡完全失去价值

### 3.4 延迟收益与立即收益

StS 有一个重要的数值原则：**延迟收益必须高于立即收益**。

| 收益类型 | 例子 | 价值系数 |
|---------|------|---------|
| 立即伤害 | Strike (6) | 1.0× |
| 下回合收益 | 格挡是即时，但易伤需要等下一击 | 需要 ~1.3× 补偿 |
| 多回合收益 | Power (能力) | 需要 2~3 回合收回成本 |
| 有条件收益 | "如果敌人有 debuff 则 +X" | 补偿 +20%~40% |

### 3.5 "膨胀"的控制

StS 开发者非常谨慎地控制数值膨胀：
- 初始卡牌是打击（6 伤害）和防御（5 护盾）
- 最顶级的攻击卡 `Bludgeon`（3 费 32 伤害）也只有打击的 5.3 倍（但 3 费）
- 经过 4 个角色 × 多轮扩展，数值天花板几乎没有提高
- **新内容的强度不是靠更高数值，而是靠更多机制交互**

## 4. 稀有度与获取概率

### 4.1 卡牌稀有度

| 稀有度 | 颜色 | 出现概率（非商店） | 出现概率（商店） |
|--------|------|------------------|----------------|
| 普通 (Common) | 灰色 | 60% | 54% |
| 罕见 (Uncommon) | 蓝色 | 37% | 37% |
| 稀有 (Rare) | 金色 | 3%（递增机制） | 9% |

**稀有卡饥饿保护机制**：
- 首次遇到稀有卡概率 = 3%（基础）
- 每次未抽出稀有卡 +1% 概率
- 抽出稀有卡后重置为 3%
- 这保证了玩家每局游戏"平均"能见到 2~3 张稀有卡

**升级卡出现概率**：
- Act 1: 0%
- Act 2: 25%（Asc 12+ 时为 12.5%）
- Act 3: 50%（Asc 12+ 时为 25%）

### 4.2 遗物稀有度

| 稀有度 | 出现权重 | 获取方式 |
|--------|---------|---------|
| 普通 (Common) | ~50% | 普通宝箱、战斗奖励 |
| 罕见 (Uncommon) | 33% | 宝箱、精英掉落 |
| 稀有 (Rare) | 17% | 宝箱、Boss 掉落 |
| Boss 遗物 | — | 仅 Boss 战 |
| 商店遗物 | — | 仅商店购买 |
| 事件遗物 | — | 仅特定事件 |

### 4.3 设计启示

1. **稀有度不是强度，而是复杂度/特异性**
2. **饥饿保护机制非常重要**——避免 RNG 让玩家一局都看不到稀有卡
3. **升级的渐进出现**——Act 1 无升级卡，让玩家在前期用白板卡学习机制

## 5. 状态/增益/减益设计

### 5.1 核心状态系统

| 状态 | 类型 | 效果 | 设计目的 |
|------|------|------|---------|
| 力量 (Strength) | 增益 | 攻击伤害 +N | 攻击系卡组的 scaling 核心 |
| 敏捷 (Dexterity) | 增益 | 护盾获得 +N | 防御系卡组的 scaling 核心 |
| 易伤 (Vulnerable) | 减益 | 受伤害 +50% | 加速战斗节奏 |
| 虚弱 (Weak) | 减益 | 造成伤害 -25% | 防守型 debuff |
| 脆弱 (Frail) | 减益 | 护盾获得 -25% | 针对防御的 debuff |
| 缠绕 (Entangled) | 减益 | 不能打出攻击牌 | 暂时封锁一种玩法 |
| 眩晕 (Stun) | 减益 | 跳过回合 | 最强控制效果，极少出现 |
| 中毒 (Poison) | 特殊 | 每回合 N 伤害，-1/回合 | 固定伤害 bypass 格挡 |

### 5.2 状态设计的黄金法则

1. **+/-25%/50% 的标准化**——大多数 buff/debuff 使用 25% 或 50% 的修正
2. **层数可叠加，但不是线性的**——Str 和 Dex 是线性，但易伤/虚弱的百分比修正不是层数叠加的
3. **每个角色有独有的 scaling 方式**——铁甲兵用力量，猎手用毒，机器人的聚焦，观者的姿态
4. **状态让白板卡牌的后期价值不会完全消失**——即使是打击，在有了力量之后也值得打出

### 5.3 状态牌（Status）设计

StS 的状态牌系统是"负面卡"设计的教科书：

| 状态牌 | 效果 | 设计目的 |
|--------|------|---------|
| 晕眩 (Dazed) | 不可打出，虚无 | 暂时污染牌组 |
| 伤口 (Wound) | 不可打出 | 污染牌组直到战斗结束 |
| 灼伤 (Burn) | 不可打出，回合结束受 2 伤害 | 污染 + 惩罚 |
| 粘液 (Slimed) | 消耗 | 至少可以打出（虽然没用） |
| 虚空 (Void) | 不可打出，抽到时 -1 能量 | 针对能量系统的惩罚 |

**设计启示**：
- 负面效果应该有**清除途径**（如 Evolve 翻伤口为抽牌、Medical Kit 可打出状态牌）
- 不同状态牌的"负面程度"不同，对应的清除方式也应不同
- 玩家应该能**与负面效果互动**，而不是单纯被惩罚

## 6. 敌人设计原则

### 6.1 敌人意图系统

StS 最优雅的设计之一是可预测的敌人意图：

- 玩家可以看到敌人下回合要做什么
- 这让玩家能做出有信息支持的决策（"这回合打满防御，下回合再输出"）
- 非随机 AI：敌人的行动模式是预设的循环/条件触发

**敌人意图类型**：
- 攻击（单体/全体，数值可见）
- 防御（获得格挡）
- Buff（给自己上增益）
- Debuff（给玩家上减益）
- 特殊（召唤、转变形态、蓄力）

### 6.2 敌人的设计层次

| 类型 | 特点 | 数量/战斗 | 设计重点 |
|------|------|----------|---------|
| 普通敌人 | 模式简单，1~2 种攻击 | 1~3 | 引入机制，考验基础攻防 |
| 精英 | 更强，有招牌技能 | 1 | 考验卡组构筑的特定方面 |
| Boss | 多阶段，独特机制 | 1 | 综合考验，需要策略应对 |

**从社区讨论中总结的设计原则**：
1. **每个精英/Boss 都应该考验不同的东西**
   - Lagavulin：考验开局爆发 vs 待机惩罚
   - Sentries：考验 AOE 清场
   - Book of Stabbing：考验高额单点防御
   - Gremlin Leader：考验召唤物处理
2. **不要设计"必须特定卡组才能过"的敌人**——这和随机卡组构建的核心理念矛盾
3. **Boss 应该有明显的 weak point**，让玩家感觉"我找到了解法"而不是"这 Boss 就是难"

## 7. 难度曲线与 Ascension 系统

### 7.1 Ascension 设计原则

Ascension 系统是 StS 在可重玩性上的杀手锏。它的 20 层难度设计遵循重要原则：

**阶梯式难度递进（每 2 层一个阶梯）**：

| 层级 | 变化 | 本质 |
|------|------|------|
| A1 | 更多精英 | 更多风险→更多奖励 |
| A2 | 普通敌人伤害+ | 数值惩罚 |
| A3 | 精英伤害+ | 数值惩罚 |
| A4 | Boss 伤害+ | 数值惩罚 |
| A5 | Boss 后回血减少 | 资源惩罚 |
| A6 | 开局受伤 | 资源惩罚 |
| A7 | 普通敌人 HP+ | 数值惩罚 |
| A8 | 精英 HP+ | 数值惩罚 |
| A9 | Boss HP+ | 数值惩罚 |
| A10 | 开局诅咒 | 牌组惩罚 |
| A11 | 少 1 药水瓶 | 资源惩罚 |
| A12 | 升级卡概率减半 | 成长惩罚 |
| A13 | Boss 金币减 25% | 经济惩罚 |
| A14 | 最大 HP 减 | 资源惩罚 |
| A15 | 坏事件变多 | 混合惩罚 |
| A16 | 商店涨价 10% | 经济惩罚 |
| A17 | 普通敌人新技能 | **机制惩罚** |
| A18 | 精英新技能 | **机制惩罚** |
| A19 | Boss 新技能 | **机制惩罚** |
| A20 | Act 3 双 Boss | **终极考验** |

**关键洞察**：
1. **前 16 层只是数值/资源惩罚**——对高手来说只是增加了难度，没有改变玩法
2. **只有 17-19 层改变了敌人的行为模式**——这才是真正的难度跃升
3. **A20 双 Boss 是最终的"综合作业"**
4. Ascension 不是单纯堆数值——低层是"用数值教育玩家需要优化"，高层是"用机制逼迫玩家思考"

### 7.2 StS 的难度曲线

```
新手期（0-10h）：适应基础，大概率死在 Act 1 Boss
成长期（10-50h）：能稳定过 Act 1，开始理解卡组构建
熟练期（50-200h）：能通关 A0，开始挑战 A20
精通期（200h+）：A20 稳定胜率，追求连胜
```

**"失败漏斗"设计**：
- Act 1 淘汰"不会基础"的玩家
- Act 2 考验"卡组是否成型"
- Act 3 考验"卡组是否足够强"
- Act 4（心臟）考验"完美执行"
- 每一层都让玩家在死前明白"我哪一步做错了"

## 8. 社区智慧与玩家观点

### 8.1 Reddit r/slaythespire 社区共识

> 以下内容综合了 Reddit、Steam 讨论区、贴吧等平台的长期讨论。

**关于好的卡牌设计**：

1. **"好的卡牌让你想做点什么"**
   - 一张好的卡牌不是"全自动变强"，而是"给了你一个新选择" 
   - 例子：`Genetic Algorithm`（每次战斗升级）→ 玩家会主动规划如何让它成长

2. **"卡牌之间的协同比单卡强度重要"**
   - 单一卡牌再强，也打不过"多张卡配合的 1+1>2"
   - 例子：`Dead Branch`+`Corruption` 的组合远强于任何单独一张稀有卡

3. **"有条件的强 > 无条件的弱"**
   - 玩家更喜欢"5 费，如果你有 X 则造成大量伤害"而不是"2 费，平平无奇"
   - 但条件不能太苛刻——不能让玩家"从来没有达成条件"

**关于平衡性**：

4. **"单卡不会毁掉平衡，组合会"**
   - StS 的开发者在采访中说过，他们不担心某一张卡太强
   - 他们担心的是**不可预期的组合**——两张"还可以"的卡放在一起变得离谱
   - 解决方式不是削弱每张卡，而是**限制组合发生的概率**（稀有度控制）

5. **"平衡不是让所有流派胜率相同"**
   - 完全平衡是不可能的，也不应该追求
   - 重要的是**每种玩法体验都有趣**，即使它不是最优解
   - 玩家能接受"毒猎手偏弱但很有特色"，不能接受"毒猎手完全是陷阱"

6. **"白板是锚点，不是终点"**
   - 打击和防御不存在"太弱"的问题——它们是**度量衡**
   - 所有其他卡牌的强度 = 白板 ± 条件/代价补偿
   - 打击/防御在后期确实弱，但这正是**玩家进步的动力**——学会删卡

**关于随机性**：

7. **"好的随机性是让你即兴创作，坏的随机性是让你无能为力"**
   - 好的：随机给你 3 张卡选 1 张（你要思考选哪个）
   - 坏的：随机给你 1 张卡不能选（你要被迫使用）
   - StS 几乎没有"坏的随机性"——即使是被动获得的状态牌，也可以通过构筑转化

8. **"随机性的关键不是公平，是有趣"**
   - 玩家不怕脸黑，怕的是脸黑时无事可做
   - 每局之间 3 选 1 的差异性 → 可重玩性
   - 只要玩家在"输之前有个有趣的过程"，输也是好的体验

### 8.2 开发者访谈摘要

**来自 Mega Crit 的 Anthony Giovannetti（2019 GDC 座谈）**：

- **关于平衡调整的方法**："我们看数据，但更重要的是我们自己玩。如果一张卡我们从不选，它就太弱了；如果每次都选，它就太强了。"
- **关于新手体验**："新手的第一需要是'感觉自己在做有意义的决策'，第二需要才是'赢'。所以我们让决策清晰可理解。"
- **关于卡牌数量**："我们曾经想加更多卡，但发现 75 张其实够了。每张卡都要有独特的生态位——如果两张卡做类似的事，为什么不让玩家少记一张？"
- **关于实验性内容**："Early Access 帮了大忙。我们放了 50 张实验性卡牌进去，看玩家怎么用。有些看起来弱的卡被玩家玩出了花，我们就保留；有些看起来强的卡被开发出了无赖连招，我们就修。"

**来自与 Reddit AMA 交流**：

- Q: "你怎么决定一张卡是普通还是罕见？"
  A: "看它的复杂性。简单效果→普通；需要一句话以上解释→罕见；核心机制改变→稀有。复杂度与稀有度挂钩，而不是与强度挂钩。"
- Q: "怎么防止数值膨胀？"

## 9. 对 Grid & Groves 的启示

### 9.1 核心差异带来的设计调整

| StS | G&G | 设计调整 |
|-----|-----|---------|
| 费用限制 | 网格格数限制 | "资源"从费用转为格子——Block 的形状/尺寸决定"价格" |
| 抽牌随机 | 手牌全可见 | 玩家可以精确定位需要放置的 Block——减少随机性，增加策略性 |
| 回合制打牌 | Bot 巡逻触发 | 触发时机的可预测性降低 -> Bot 路径操控成为核心技巧 |
| 药水/遗物 | Stat 系统 | Stat 替代了大部分遗物和药水的功能 |
| 删牌商店 | 卡组大小限制 | 卡组固定上限（10~50），强制玩家精简化 |

### 9.2 可以借鉴的设计原则

1. **"稀有度=复杂度"原则**
   - 普通 Block：简单效果（纯伤害/纯护盾）
   - 稀有 Block：带 1 个机制（如松动/过载）
   - 史诗 Block：复合机制（如触发相邻 Block）
   - 传说 Block：改变游戏规则（如改变 Bot 路径）

2. **"饥饿保护"原则**
   - 玩家应该在前 N 场战斗中"保底"见到若干稀有 Block
   - 可以借鉴 StS 的递增保底机制

3. **"负面互动的可能性"原则**
   - 状态牌（粘液、灼伤等）看起来是纯粹惩罚
   - 但提供了与它们互动的方式（Evolve Fire Breathing Ironclad 等）
   - G&G 中也可以设计"与其他机制联动"的负面效果

4. **"延迟收益溢价"原则**
   - 驻留 Block（永久占格）：单次触发数值应接近或略高于立即触发的 Block，补偿不可适配风险
   - Stat 积累（跨战斗保留）：触发时数值应显著低于直接伤害
   - 立即收益（一次性/松动）：单次较高以补偿损失

5. **"让每个决策都有权衡"原则**
   - 没有无脑强的 Block
   - 每个 Block 应该在"伤害/护盾/机制/格子效率"上有取舍
   - 例子：一个 2×2 的 Block 总伤害高，但占用 4 格 = 不到总格数 35 的 1/8

6. **"对战 Boss 应该有明确策略"原则**
   - Boss 应该有可观察的意图（已在 G&G 中实现的敌人意图系统）
   - 每个 Boss 应该考验特定的能力（爆发/续航/防御/清杂）
   - 避免"只有特定卡组才能过"的 Boss 设计

### 9.3 具体的数值建议

**基于 StS 模型，G&G 的数值锚点建议**：

```
1格（1×1）白板伤害 = 6  ← 和 StS 的 Strike 一致，这是锚点
1格（1×1）白板护盾 = 5~6

每增加 1 格（多部件），总伤害约 ×1.4~1.6（因为占多格代价高）
松动 Block: ×1.2（提前离场补偿——本回合不能复用）
驻留 Block: ×1.1（永久占格不可移除补偿）
一次性 Exhaust: ×1.3~1.4（单次补偿）
Stat 给予 Block: 本身 0 伤害/护盾（Stat 是跨战斗收益）
```

**Stat 系统建议**（参考 StS 的遗物流）：

- 战斗内 Stat（每场清零）：可以相对强，因为需要每局重新积累
  - 例如："本场战斗中每放置 1 个 Block，伤害 +1"
- 跨战斗 Stat（全局保留）：必须严格稀有限量
  - 参考 StS 遗物数量——一场完整的通关大约获得 10~15 个遗物
  - 所以 G&G 的一整局游戏大约获得 10~15 个跨战斗 Stat
  - 每个 Stat 应该有明确的获取条件（Boss 战、商店、特定事件）

### 9.4 难度系统建议（参考 Ascension）

借鉴 Ascension 的分层设计，G&G 也可以设计类似系统：

| 层级 | 变化 |
|------|------|
| +1 | 敌人伤害 +10% |
| +2 | 网格缩小 1 列（6 列） |
| +3 | Boss 获得额外技能 |
| +4 | 商店涨价 20% |
| +5 | 初始卡组 -1 打击（3 打击 + 4 防御） |
| +6 | 敌人出现频率 +30% |
| +7 | 精英敌人新增模式 |
| +8 | 网格缩小 1 行（6×5） |
| +9 | Boss 战后回血减少 |
| +10 | 开局 1 个 Block 被"封禁"（本轮不可用） |

> 核心：前几层是数值/资源惩罚，后几层是**机制改变**。
