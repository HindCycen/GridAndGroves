# 数值平衡框架

> 参考《杀戮尖塔》(Slay the Spire) 卡牌数值模型，结合 Grid & Groves 网格放置机制建立的平衡锚点。

---

## 1. 游戏完整循环与经济系统

### 全局流程

```
开局
  → 选择角色 → 选择 1 个主卡包（30~35 Block） + 系统随机抽 4 个小卡包
  → 获得初始卡组: 4 打击 + 4 防御 + 1 角色固有 + 1 角色能力
  → 进入游戏，持有 10 金币

战斗
  → 玩家在网格上放置 Block
  → End Turn → Bot 巡逻触发 Block
  → 敌人行动
  → 重复直到胜负

战后
  → 胜利: 获得金币 = 战斗难度基准 + Rand(-1, 0, 1)
  → 进入商店

商店规则
  ├─ 可购买 Block（从卡池中随机展示若干）
  ├─ 可重掷商品（费用需要约定）
  ├─ 可"售卖"按钮: 卖出当前卡组中的 Block
  │   ├─ 打击/防御: 售价 1 金币
  │   └─ 其他: 售价 = 买入价 × 1/2
  ├─ 离店检查:
  │   ├─ 卡组 < 10: 自动补打击/防御到 10 张（尽量均衡，不能均衡时防御更多）
  │   └─ 卡组 > 10: 不允许离店，必须售卖到 ≤ 10
  └─ Boss 战后商店:
      ├─ 多售卖 1 件商品
      ├─ 所有商品价格 = 0 金币
      └─ 不可重掷
```

### 初始卡组构成

| 槽位 | 数量 | 说明 |
|------|------|------|
| 打击 (Strike) | 4 | 1×1 普通伤害 6 |
| 防御 (Defend) | 4 | 1×1 普通护盾 6 |
| 角色固有 (Character Innate) | 1 | 类似 StS 的痛击/中和/生存者——每个角色的标志性 Block |
| 角色能力 (Character Ability) | 1 | 类似 StS 的初始遗物——本游戏中没有独立的遗物/药水系统，所有"遗物/药水"等价于一个触发后立即从牌组和战斗中移除的 Block |

> **打击和防御**是白板基准，所有其他 Block 的强度都应与它们比较。
> 打击 = 1×1 伤害 6 = 1.0 价值单位；防御 = 1×1 护盾 6 = 1.0 价值单位。

## 2. 核心差异：没有"费用"和"抽牌"概念

### 本游戏与 StS 的关键区别

| 维度 | StS | Grid & Groves |
|------|-----|---------------|
| **费用** | 每张牌有 1~3 费 | **无费用**——任何 Block 都可以放置，唯一的限制是网格空闲格数 |
| **抽牌** | 每回合抽固定张数，有检索困难 | **无抽牌限制**——牌组很小（10~50），展示区可浏览全部手牌，玩家可定向检索 |
| **能量** | 每回合 3~5 费总上限 | **无能量**——你可以在空闲格上放任意多 Block |
| **资源限制** | 费用 + 抽牌 | **网格格子**（7×5 = 35，受敌方 Block 挤压）|

### 这并不意味着没有"运转"

"抽牌"和"加费"概念不存在，但游戏的成长弧线仍然需要**阶梯式上升**：

- 前期: 靠白板打击(6)/防御(6) 可以应付
- 中期: 光靠白板每格 6 点效率不足以应对强敌
- 后期: 玩家必须通过以下两种思路之一来 scaling

## 3. 两条 Scaling 路径

### 路径 A：跨战斗 Stat 积累

Stat（属性/状态）**跨战斗保留**，玩家可以通过叠加 Stat 来提升自己的攻防强度。

```
战斗 1: 获得"力量 +2" Stat → 战斗 2: 所有伤害 +2
       → 获得"藤蔓"Stat → 战斗 3: 每回合对敌人造成 X 伤害
       → 叠加 → 后期 Stat 层数极高 → 碾压
```

**设计原则**：
- ❌ Stat 不应急于提高白板数值——Stat 是跨战斗积累的，长期收益极高
- ❌ 不要让 Stat 的获取过于泛滥——这会让"无 Stat 的纯白板卡组"无法生存
- ✅ 类比 StS **遗物**——Stat-granting Block 应该是稀有的、需要策略选择的亮点，而非人手一份的基础
- ✅ 在 G&G 中，"遗物"= 一个使用后立即从牌组和战斗中移除的、给玩家添加一个 Stat 的 Block

### 路径 B：Bot 路径操控

Bot 的巡逻方向可以被 Block 的 `MovingDirection` 改变。熟练的玩家可以通过放置方向修改 Block，让 Bot **折返**、**跳跃**、**循环**，从而在一回合内多次触发同一格上的 Block。

```
蛇形路径（默认）: Bot 从左上 → 蛇形 → 右下，35 tick
路径操控后      : Bot 在特定列折返上行 → 再次经过已经触发的 Block → 复用格子
极端情况        : 构造循环路径 → Bot 永远在同一行来回走 → 无限触发
```

**设计原则**：
- ⚠️ **所有有利或达成无限的 Block，其白板数值应当降低**——因为"每回合触发一次"变成了"每回合触发 N 次"，单次伤害必须低才能防止失控
- ✅ 路径操控是高手向的进阶玩法，不应成为通关的唯一路径
- ✅ 提供"路径操控收益"的 Block 应当附带明显的代价或低白板数值

### 两条路径的数值影响总表

| Block 类型 | 单次触发价值 | 每回合潜在总价值 | 原因 |
|-----------|------------|----------------|------|
| 白板打击/防御 | 6 | 6（触发 1 次） | 基准 |
| 松动 Block | 7 (基准 ×1.2) | 7（触发 1 次，提前离场） | 提前离场补偿——本回合不能复用 |
| 一次性 Exhaust | 8~9 (基准 ×1.4) | 8~9（只触发 1 次） | 单次补偿 |
| 驻留 Block | 6~7 (基准 ×1.1) | 6~7 × 剩余回合数 | 永久占格不可移除，补偿不可适配风险 |
| 给玩家添加 Stat 的 Block | 0（无直接伤害） | 跨战斗保留 | 当作遗物设计，Stat 不可泛滥 |

## 8. 稀有度乘数

| 稀有度 | 伤害倍率 | 护盾倍率 | 说明 |
|--------|---------|---------|------|
| 普通 | ×1.0 | ×1.0 | 白板基准 |
| 稀有 | ×1.3~1.5 | ×1.3~1.5 | 比普通强 30~50% |
| 史诗 | ×1.8~2.2 | ×1.6~2.0 | 比普通强 80~120% |
| 传说 | ×2.5~3.0+ | ×2.0~2.5 | 改变局势 |

> 倍率应用于单部件基准值。例如稀有伤害 = 6 × 1.4 ≈ 8~9。

## 9. 形状效率与取舍

### 每格每回合价值密度

| 形状 | 占用格数 | 普通总价值 | 价值密度 | 取舍 |
|------|---------|-----------|---------|------|
| 1×1 | 1 | 6 伤 / 6 盾 | **6.0** | 最灵活，但总伤低 |
| 1×2 横 | 2 | 8~10 伤 | 4.0~5.0 | Bot 蛇形路径容易覆盖 |
| 2×1 竖 | 2 | 8~10 伤 | 4.0~5.0 | 纵列放置受限 |
| L 形 | 3 | 12~15 伤 | 4.0~5.0 | 占 3 格，爆发高 |
| 2×2 | 4 | 12 伤 | 3.0 | 占 4 格效率最低，总伤最高 |

### 关键平衡结论

1. **1×1 的价值密度最高**——这正是为什么单格 Block 是卡组基石。
2. **大形状的总价值更高但密度更低**——但大形状的优势在于"一次放置、一次触发、不需要吃多次手牌位"。
3. **松动机制需要数值补偿**——松动 Block 提前离场，本回合不能复用，释放的格子在中途无实际收益。因此松动 Block 应获得 ×1.15~1.3 的数值补偿。

## 12. 实战计算公式

### Block 总价值估算

```
总价值 = Σ(部件伤害 × 稀有度倍率 × 形状系数 × 标签系数)

形状系数:
  1×1: 1.0
  1×2: 0.9（每个部件）
  2×1: 0.9
  L 形: 0.85
  2×2: 0.8

标签系数:
  无标签:     1.0
  一次性:     1.3~1.5
  松动:       1.15~1.3
  驻留:       1.05~1.15
  无限循环（非松动）: 0.4~0.5
  过载消费:   0.8 + 每层 1 额外
  共生:       0.8 + 场上 Block 数 × 1~2
  共鸣:       0.9 + 链位置 × 1
```

### 示例验算

```
"废料洪流"（稀有，1×2 横，松动）
  部件 A 7 × 1.4 × 0.9 × 1.2 = 10.6
  部件 B 5 × 1.4 × 0.9 × 1.2 = 7.6
  总 ≈ 18.2，占 2 格，松动提前离场补偿 ≈ 合理

"铁砧"（稀有，2×2，一次性）
  4 部件 × 3 × 1.4 × 0.8 × 1.4 = 4 × 4.7 = 18.8
  总 ≈ 19，占 4 格一次性爆发 ≈ 合理

"千年古树"（史诗，2×2，扎根）
  4 部件 × (7+5+4+5) = 21 / 回合
  驻留 1.1 × 史诗 2.0 ≈ 46.2 每回合等效
  占 4 格永久不能复用，不可移除风险补偿 ≈ 合理
```

## 13. "遗物类"Block 设计准则

本游戏没有独立的遗物/药水系统。"遗物效果"= **给玩家添加一个跨战斗保留的 Stat，然后自身从牌组和战斗中移除的 Block**。

### 设计原则

| 原则 | 说明 |
|------|------|
| **稀有** | 每个角色最多 3~5 个此类 Block，不应泛滥 |
| **一次性** | 触发后 Exhaust 并**从牌组中永久移除**（不会在后续战斗中洗入） |
| **效果永久** | 添加的 Stat 跨战斗保留，直到游戏结束或特定条件移除 |
| **数值保守** | +2 力量永久≈ StS 手里剑级别，不应出现在普通 Block 中 |
| **有代价** | 可以附带血量代价或卡组位置代价 |

### 示例

```
"旧世界芯片"（Old World Chip）
  1×1 @ (0,0)，方向向下
  行为: GrantPlayerStatBehavior → 添加 StatDef "永久力量+1"
  效果: 触发后 Exhaust，从本局卡组中永久移除
  稀有度: 史诗

"生物样本"（Bio Sample）
  1×1 @ (0,0)，方向向下
  行为: GrantPlayerStatBehavior → 添加 StatDef "每回合开始获得 2 护盾"
  效果: 触发后 Exhaust，从本局卡组中永久移除
  稀有度: 史诗
```

## 14. 角色固有 + 角色能力设计框架

### 角色固有 (Character Innate)

对应位置：初始卡组的第 9 张。类似 StS 的痛击(铁甲战士)/中和(静默猎人)/生存者(故障机器人)。

| 角色 | 固有 Block | 定位 |
|------|-----------|------|
| 铁锈游侠 | "锈蚀核心" (Rust Core) | 1×1，效果: 伤害 4 + 施加 2 层锈蚀 + 松动 |
| 星语术士 | "共鸣核心" (Resonance Core) | 1×1，效果: 伤害 4 + 共鸣标签 + 抽 1 Block |
| 翠绿哨兵 | "共生核心" (Symbiosis Core) | 1×1，效果: 护盾 5 + 共生（每 Block +1） + 扎根 |

### 角色能力 (Character Ability)

对应位置：初始卡组的第 10 张。类似 StS 的燃烧之血(铁甲战士)/蛇戒(静默猎人)/裂缝(故障机器人)。

在本游戏中，角色能力 = 一个 1×1 的 Exhaust + 永久移除的 Block，触发后给玩家添加一个全局 Stat。

| 角色 | 能力 Block | 效果 |
|------|-----------|------|
| 铁锈游侠 | "废品回收协议" | 永久: 每回合第 1 个松动 Block 被触发时，抽 1 Block |
| 星语术士 | "星能回路" | 永久: 共鸣链最大长度从 3 增加到 4 |
| 翠绿哨兵 | "光合纲领" | 永久: 每回合开始获得 2 护盾 |
